UMA REDE BARATA – CABO COAXIAL

(Para uma pequena e mais confiável rede veja a de par trançado abaixo.)

A rede com cabo Coaxial é um dos tipos de rede mais econômica, mas também o mais instável devido ao fato deste tipo de cabo não ser muito flexível e partir com freqüência.

A impedância (resistência à passagem de corrente) nos cabos coaxiais para Placas de Rede tipo Ethernet é de 50 ohms.

VANTAGENS DA REDE COM CABOS COAXIAIS:

- Sua blindagem permite longos comprimentos

- Permite redes multicanal

- Mais barata que a de par trançado blindado

DESVANTAGENS:

- Quebra e apresenta mau contato facilmente (devido a falta de flexibilidade)

- Difícil manuseio

- Mais caro que o de par trançado não blindado

COMPONENTES:

- Placas de Rede compatível com NE2000 (sugestão)

- Cabos coaxiais

- Conector BNC soldável

- Tê

- Terminador

- Conector BNC para Crimp (ou por pressão) – Este não é soldável, para sua conexão utiliza-se um alicate de pressão apropriado (crimp).

- Descascador de cabo

- Alicate para crimp

COMO SABER SE FIO CENTRAL ESTÁ PARTIDO:

- Ajustar o multímetro (VOM) para resistência

- Colocar uma das pontas do VOM em contato com uma das pontas do fio e a outra ponta do VOM em contato com a outra do fio.

- Caso esteja apartado o ponteiro do VOM não mexe

- Caso contrário o fio está OK.

Obs.: muitas vezes o cabo parte-se exatamente na junta com o Conector BNC.

 

 

 

 

REDE WINDOWS 95/98 COM CABO DE PAR TRANÇADO

Para pequenas redes podemos utilizar o próprio Windows 95/98 e um Hub com cabos de par trançado (RJ 45).

HARDWARE

MATERIAL NECESSÁRIO

- Pacas de Rede Ethernet (10 ou 100 MBits)

- Hub para 8 ou mais micros, dependendo

- Cabo de par trançado (RJ45)

- Conectores RJ45. 2 para cada micro.

ESQUEMA DE LIGAÇÃO

 

 

 

 

 

 

 

 

 

 

 

Veja que cada micro se liga ao Hub, de maneira que mesmo que alguns estejam com problemas os que não estão possam se comunicar.

Existem também Tês para o RJ45 como alternativa para o Hub no caso de apenas 2 ou 3 micros, ou ainda de uma rede de bem mais baixo custo de inicial e mais alto custo de manutenção.

SOFTWARE

Obs.: Sempre devemos lembrar de, ao comprar as Placas, exigir os drivers (Disquete ou CD)

CONFIGURANDO A REDE

Painel de Controle – Rede

Devemos ter os seguintes itens:

- Cliente para redes Microsoft

- Adaptador para Rede Dial-Up

- NetBEUI (Protocolo – Microsoft)

- IPX/SPX (Protocolo – Microsoft)

- TCP/IP (Protocolo – Microsoft)

- Compartilhamento de arquivos e impressoras para rede Microsoft (Serviços)

Ainda na tela de configuração da Rede certifique-se de que o

Logon primário da rede seja o

Logon do Windows

 

Como todos os micros passam pelo Hub, qualquer micro que tenha sua conexão com problema deixará de acender o LED do Hub, tornando fácil a identificação do micro

Com problema na rede.

Cuidado ainda para não esquecer desligada a fonte do Hub.

 

 

REDE COMPARTILHANDO UMA ÚNICA LINHA TELEFÔNICA

E UM MODEM COM OS CLIENTES

VIA WINDOWS98SE

 

INSTALAÇÃO DO SOFTWARE NO MICRO QUE SERÁ SERVIDOR

Obrigatoriamente deve ser o Windows98SE ou superior.

- Painel de Controle – Adicionar – Instalação do Windows – Ferramentas de Internet – Internet Connect Sharing

- Será pedido o disco de instalação do Windows.

- Após instalar será aberto um Assistente, na primeira tela clique em Avançar.

- Escolha o tipo de conexão (a default é a mais comum: modem) e Avançar.

- Selecione a conexão que utilizará para acessar a Internet e Avançar.

- Coloque um disquete no drive para que seja copiado o arquivo a ser instalado nas máquinas Clientes.

- Será sugerida a reinicialização do micro. Reinicie.

 

CONFIGURAÇÃO DOS CLIENTES:

- Execute o arquivo "icsclset.exe" do disquete.

Simplesmente mande Avançar até concluir.

Não esqueça de entrar na rede (com o login e a senha correta).

Experimente conectar na Internet pelo servidor e acessar pelos clientes. Você agora pode navegar na Internet em todos os micros da rede e também passar e receber e-mail pelos clientes de e-mail.

Caso antes houvesse alguma configuração do protocolo TCP/IP dê uma olhada no arquivo Readme.txt ou Leiame.txt do disquete para os devidos ajustes.

 

 

 

UTILITÁRIOS PARA REDES

SOFTWARE DE COMUNICAÇÃO LOCAL NA REDE

O Windows 95 ou 98 já traz um programa para troca de mensagens de micros ligados em rede. É o winpopup.exe. Ele encontra-se na pasta c:\windows.

Como ele está no "path", basta que o chamemos de qualquer lugar:

Iniciar – Executar – winpopup

O ideal, para quem vai utilizar com frequência é criar um atalho para o mesmo, lembrando o caminho completo: c:\windows\winpopup.exe.

MEDINDO O DESEMPENHO DA REDE

System Monitor – este vem com o windows 95 ou 98.

CONCEITOS LIGADOS A REDES

IP

- Números de 32 bits, escrito em 4 partes separadas por ponto, como 127.7.0.125

- A primeira parte identifica uma rede na Inter-rede

- A Segunda parte identifica um host dentro da rede

- A terceira eu gostaria de saber a função :)

- A Quarta identifica cada estação da rede

127.0.0.0 – este é um IP especial, utilizado para testes e para a comunicação entre processos de uma máquina.

MÁSCARA – Em redes locais o Host (servidor) recebe um ID (código de Identificação) exclusivo e todas as estações (clientes) terão a máscara 255.255.255.0.

DNS – Domain Name System ?

SMTP – Simple Mail Transfer Protocol – É o protocolo utilizado para ENVIAR as mensagens de e-mail (pelos programas clientes de e-mail como Outlock e Netscape Messager)

POP – Protocolo utilizado para RECEBER as mensagens de e-mail pelos programas clientes de e-mail.

PROXY - ?

FTP – File Transfer Protocol – Procolo de transferência de arquivos. Vários sites tem uma interface tipo o Explorer, com pastas e arquivos. Isso caracteriza os sites de FTP, onde existem apenas arquivos para download (baixar) ou upload (enviar). Existem programas expecíficos para o envio (upload) e recebimentos de arquivos (download), como o WS FTP, o CUT FTP e outros. A transferência de arquivos por sites de FTP é sensivelmente mas rápida do que por sites de HTTP, já que os de FTP são especializados no assunto e não tem as "gorduras" dos sites de HTTP (as páginas convencionais).

HTTP – Hiper Text Transfer Protocol ?

TCP/IP – A arquitetura da Internet é baseada nos dois protocolos: TCP e IP

TCP – Transmission Control Protocol

IP – Internet Protocol

TELNET – Protocolo que permite que um usuário remoto, através de uma sessão de Terminal, opere como se estivesse no próprio computador distante.

CGI – Common Gatway Interface – Permite interação em páginas WWW, como instrumentos de busca, formulários, etc.

JAVA – Linguagem de programação, que quando interpretada por um browser, não tem permissão para gravar dados no disco rígido, sendo em con sequência uma linguagem segura para o computador do usuário.

JAVA Script – Uma linguagem adaptada do Java, mais simples, possibilitando o uso em páginas até por quem não tem conhecimento de Java.

APPLETS – Programa escrito em Java, inserido em uma página html.

PLUGINS – São módulos que podem ser adicionados ao browser com recursos extras, como vídeo, som, pdf, etc.

URL – Uniform Resource Locator – Endereço exclusivo de recursos da Internet: páginas http, ftp, telnet, etc.

WWW – World Wide Web – Grande Rede Muldial. Conjunto de documentos da Internet (em html).

LAN – Local Area Network – Rede de alcance local.

SERVIDOR – Computador que eleva a capacidade de processamento, com a função de oferecer serviços à rede. Ficam normalmente em sistemas operacionais mais especializados para rede e potentes e também em máquinas mais robustas.

IPX/SPX – Internet Package eXchange/Sequence Package eXchange. Usado em redes netware de pequeno e médio porte e tem suporte a roteamento.

NET BEUI – Network Basic End User Interface – Pequenas LAN’s, é rápida e simples.
